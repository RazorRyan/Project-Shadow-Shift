# Shadow Shift Phaser V2 Prompt Pack

## What this is
- A low-token execution pack for evolving the current Phaser V2 sandbox into a scalable Metroidvania framework.
- Built for short operator-led runs: inspect, plan briefly, implement, validate, stop.

## How to use it
1. Run one phase file only.
2. Keep scope to the first working slice.
3. Test immediately after the phase stops.
4. Commit before moving forward.
5. Use bundle files only as guided run groups, not batch execution.

## Phase order
- Run `01_phase_foundation_audit.md` through `27_phase_polish.md` in order.
- Do not skip phases.
- Do not auto-chain phases.

## Execution rules
- Minimal output only.
- Code changes over explanation.
- Stop after each phase.
- Fix failures before continuing.

## Commit rule
- Commit after each passing phase.

## First run
- Start with `01_phase_foundation_audit.md`.
